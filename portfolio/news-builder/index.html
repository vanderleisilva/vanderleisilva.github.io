News Builder Newsletter interface builder

Client

A big marketing agency has shown the need of a creation of a module on its self made Enterprise resource planning system (ERP) for an easy creation of newsletters, up to that point, every newsletter was created by a third party.

The brief

The currently client system was based in a self made php framework then the solution needed follows its good practices in terms of coding and interface. For that goal Iâ€™ve worked with a third party IT professional, currently responsible for the ERP. For this project the deadline was 2 months with weekly short releases.

The solution

The News Builder module receives a compressed file with the HTML newsletter base, with special keywords where the dynamic content will be inserted, then based on it the module creates a drag and drop interface that the end user can manage its content, images and structure.

This implementation has used PHP for backend and jQuery for frontend and its communication based on api rests. All coding developed following object calisthenics concepts.
